09_Python多进程和多线程

1 多任务

对于操作系统来说，一个任务就是一个进程（Process），比如打开一个浏览器就是启动一个浏览器进程，打开两个记事本就启动了两个记事本进程，

有些进程内部要同时干多件事，比如Word同时进行打字和拼写检查，需要同时运行多个“子任务”，我们把进程内的这些“子任务”称为线程（Thread）。线程是最小执行单元，一个进程至少有一个线程。

  • 串行：多个任务依次执行
  • 并发：对于单核CPU，多个任务快速切换执行，看起来好像是多个任务一起执行的
  • 并行：对于多核CPU，真正实现了多个任务同时执行

多进程和多线程由操作系统进行调度，执行顺序具有随机性。

2 多进程

2.1 multiprocessing

Windows可以使用跨平台的 multiprocessing，模块中提供了 Process类代表一个进程对象。

  • target：要执行的函数名（注意没有 ()，否则会直接调用）
  • [name]：为进程起的名字，可以通过 进程对象.name访问
  • 参数：为函数所需参数传值
    • args：传入元组，按照位置参数进行传值
    • kwargs：传入字典，按照键值对进行匹配传参

2.2 获取进程编号

每个程序（进程）在设备上都有唯一的进程编号（PID），程序释放时进程编号也会释放，实现了进程编号的重复利用。

  • 获取当前PID的方式：
    • 方式一：os.getpid()
    • 方式二：multiprocessing.current_process().pid
  • 获取父进程PID的方式：
    • os.getppid()

在 main中创建的进程，如果没有指定，则它的父进程都是 main进程，而 main的父进程是 Pycharm程序。

2.3 数据隔离

多进程有以下特点：

  1. 进程之间执行是无序的
  2. 进程之间有数据隔离
  3. 主进程会等待所有子进程执行结束再结束

每个进程都有自己独立的内存空间，进程之间的数据是相互隔离的，子进程中的资源是父进程资源的拷贝，互不影响。

2.4 守护进程

注意这个词的描述非常不准确。 正常来说，主进程会等待所有子进程执行结束之后才结束，但是有时候想要关闭主进程的同时关闭所有子进程，这个时候需要守护进程。

如果想要实现子进程和主进程同步关闭，有两种实现方法：

  1. 设置子进程为守护进程，将属性 daemon设为 True
  2. 手动杀死子进程，不过会导致子进程变为僵尸进程，会交由Python解释器自动回收

2.5 Manager实现数据共享

在多进程中数据共享可以使用 multiprocessing模块中的 Manager()。

3 多线程

3.1 threading

多任务可以由多进程完成，也可以由一个进程内的多线程完成。进程是CPU分配资源的基本单位，线程是CPU调度资源的最小单位，线程是依附于进程的，一个进程至少含有一个线程。

多线程之间有以下特点：

  1. 线程之间执行是无序的
  2. 统一进程下的多线程之间共享全局变量
  3. 主线程会等待所有子线程执行结束再结束

使用 threading模块中的 Thread类创建线程对象。

  • target：要执行的函数名（注意没有 ()，否则会直接调用）
  • [name]：为线程起的名字，可以通过 线程对象.name访问
  • 参数：为函数所需参数传值
    • args：传入元组，按照位置参数进行传值
    • kwargs：传入字典，按照键值对进行匹配传参

3.2 数据共享

多线程之间共用内存空间，数据是共享的。

3.3 守护线程

和进程类似，主线程会等待所有子线程结束再结束，如果想要让子线程随主进程结束而结束，应该设置守护线程，线程.daemon = True。

3.4 互斥锁

多线程中所有变量都由所有线程共享，最大的危险在于多个线程同时修改同一变量，内容就会出现紊乱。

在上述案例中，num += 1并不是原子操作，而是分为以下几步交由CPU执行：

  1. 读取当前 num 的值到寄存器
  2. 将寄存器中的值加1
  3. 将结果写回 num 变量

为了确保计算正确，就要给两个 add()函数上个锁，当某个线程获得了该锁，其他线程就不能同时执行，而要等待直到获得该锁的线程释放锁，并获得该锁。由于锁只有一个，同一时间只有一个线程可以持有该锁，就不会造成修改的冲突。

通过 threading.Lock()可以创建一个锁对象。

当多个线程同时执行 lock.acquire()时，只有一个线程能成功地获取锁，然后继续执行代码，其他线程就继续等待直到获得锁为止。

获得锁的线程用完后一定要释放锁，否则那些苦苦等待锁的线程将永远等待下去，成为死线程。所以我们用 try...finally来确保锁一定会被释放。

Python解释器由于设计时有GIL全局锁，导致了多线程无法利用多核。多线程的并发在Python中就是一个美丽的梦。

4 进程和线程的区别

  1. 线程是依附于进程的，没有进程就没有线程
  2. 一个进程最少有一个线程，进程可以创建多个线程
  3. 进程之间数据隔离，线程之间共享全局变量（使用互斥锁解决资源竞争问题）
  4. 进程是操作系统分配资源的基本单位，线程是CPU调度的基本单位
  5. 进程更消耗资源，相对更稳定；线程更轻量级，相对更灵活
