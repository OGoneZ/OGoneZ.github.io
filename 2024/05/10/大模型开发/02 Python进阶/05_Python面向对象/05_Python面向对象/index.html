05_Python面向对象

面向对象就是老板思想，需要什么工作就派给什么职位（类）的员工，对于相同的职位都有相同的特征，每一种职位都有很多不同的人（对象）在岗位上，而每一个人又有自己的特点（属性）和技能（方法）。

1 类和对象

  • 类：抽象的模板，是对一群有相同特征或行为的事物的统称，不能直接使用。（比如学生类）
    • 属性：这类事物具有什么样的特征
    • 方法：这类事物具有什么样的行为
  • 对象：具体的实体，是由类创建出来的一个具体存在，可以直接使用。（比如张三）

1.1 定义类和创建对象

在Python中，函数是独立的代码块，可以在任何地方定义和调用。方法是定义在类内部的函数，通常用于操作类的实例或类本身。

self关键字：类似Java的this 为了在类的内部区分是哪个对象调用了类的方法，哪一个对象调用的方法，self就是哪一个对象的引用。调用方法时，不需要传递self参数。
在方法内部：

  • 通过self.访问对象的属性
  • 通过self.调用对象的其他方法

实例方法的第一个参数必须为self！

1.2 属性的增加与访问

与Java不同的是，Python可以在类的外面直接添加属性，但不推荐这种方式，属性应该封装到类的内部，参考下一节的__init__方法。

由于这种方法的存在，和静态语言不同，Python允许对实例变量绑定任何数据，也就是说，同一个类的不同实例，拥有的变量名称可能不同

1.3 获取对象信息

dir(对象)可以查看该对象有哪些属性和方法。

使用dir()只能把属性和方法列出来，搭配getattr()、setattr()以及hasattr()可以改变对象的属性和方法。

  • hasattr()查看是否有指定属性
  • getattr()获取指定属性
  • setattr()设置指定属性

只有在不知道对象信息的时候，我们才会去获取对象信息，否则直接使用对象.属性即可。

1.4 身份运算符is、is not

身份运算符is、is not用于比较两个对象的内存地址是否一致——是否是对同一个对象的引用。

  • is用于判断两个变量引用对象是否为同一个
  • ==用于判断引用变量的值是否相等

针对None的比较，建议使用is。

2 魔法方法

魔法方法使用__xx__双下划线包围，在特殊情况下会自动调用。

2.1 __init__方法

__init__就是Python的构造方法，也用来声明实例属性，创建对象时自动触发。

2.2 __str__方法

使用print()打印对象时，默认是打印对象的内存地址，如果不想要输出内存地址，就需要定义__str__方法。__str__方法的返回值必须是字符串，打印时将会打印其返回值。

2.3 __repr__方法

  • __str__：面向用户，强调可读性，在print时会调用
  • __repr__：面向开发者，强调更准确、更标准的对象表示形式，通常是调试用的，交互式解释器中会调用

如果只写 __repr__ 而没写 __str__，那 print(obj) 也会退回调用 __repr__。

2.4 __del__方法

当删除对象时，会自动调用__del__方法，让对象在销毁前做一些事情。

2.5 __len__方法

定义调用len(对象)函数时，返回的长度值。

2.6 __iter__方法

通过实现__iter__方法，使用yield关键字，可以使自定义类成为可迭代对象，方便使用for循环进行遍历。

3 封装

根据职责将属性和方法封装到一个抽象的类中，外界使用类创建对象，让对象使用属性、调用方法，对象方法的细节都被封装在类的内部。

私有属性和私有方法只能在类的内部使用，在类的外部使用需要通过公共接口。

3.1 私有属性

格式：在属性名前面加两个下划线__。 用法：和Java一样，定义公有方法get_xxx获取私有属性、set_xxx设置私有属性。

3.2 私有方法

格式：在方法名前面加两个下划线__。 用法：定义公有方法调用私有方法。

了解：其实Python中没有真正的私有，在外部使用类名.__属性、类名.__方法也可以访问，但不要这样做。 实际上Python内部只是将私有成员改了一个名，__属性变成了_类名__属性，__方法变成了_类名__方法。

3.3 __dict__属性

___dict___ 是 Python 中的一个特殊属性，用于存储对象的所有属性及其对应的值。它以字典形式呈现，键为属性名，值为属性值。

在文件写入中可以使用__dict__将对象转为字典，再将字典转为字符串进行写入。

4 继承

4.1 继承的定义

  • 父类：也叫作基类
  • 子类：也叫作派生类或扩展类

子类默认继承父类所有的属性和方法，Python 的子类不会直接继承父类的私有属性和私有方法。

当多个父类中存在相同名字的属性或方法，默认使用第一个父类的同名属性和方法，可以使用__mro__属性或mro()方法查看先后顺序。 MRO(Method Resolution Order)：方法解析顺序

在Python中，object是所有类的基类，即所有Python对象的根源。如果没有指定父类，会默认使用object作为该类的基类（可以使用dir函数查看）。

4.2 重写

父类方法不满足子类需求时，可以重写（override），直接在子类中使用和父类相同名字的属性和方法即可进行重写。重写之后会调用子类的方法，而不是父类的。

4.2.1 “覆盖”父类

子类方法和父类方法完全不同，在子类中写一个和父类同名的方法实现即可。

4.2.2 “扩展”父类

方式一：父类名.父类方法(self)

这种方法可以精准访问指定的父类。

方式二：super().父类方法()（更常用）

super() 实际上返回的是一个 代理对象（super object）。这个代理对象知道两件事：

  1. 当前是哪个类（__class__）
  2. 当前实例的 MRO 是什么

只能访问最近的父类，有就用，没有就往后找，找不到再报错。

super()零参数时，Python 解释器会自动填充两个参数：

  • __class__：当前正在定义的类。
  • self：当前实例。

在Python中super是一个特殊的类，super()是使用super类创建出来的对象。使⽤super()可以⾃动查找⽗类，适合单继承使⽤，多继承不建议使用。

5 多态

5.1 多态的定义

当我们定义一个class的时候，我们实际上就定义了一种数据类型，判断一个变量是否是某个类型可以用isinstance()判断。在继承关系中，如果一个实例的数据类型是某个子类，那它的数据类型也可以被看做是父类。但是，反过来就不行。

多态指的是同一函数对于传入的不同对象做出不同的响应。多态的好处在于，当我们需要传入Dog、Cat等等时，我们只需要接收Animal这个父类类型就可以，并且原有的函数对于新增的Animal子类也生效，这就是多态的含义。

实现多态的条件：

  1. 有继承（子类继承父类）
  2. 有方法重写（子类重写父类方法）
  3. 父类引用指向子类对象（子类对象传给父类对象调用者）

对于静态语言Java来说，多态需要继承同一父类或接口，在刚刚的例子中必须是Animal或其子类。而在Python中，不关心对象的类型，只关心对象的行为（即它有什么方法）。只要一个对象具有所需的方法和属性，它就可以被当作预期的类型来使用，在刚刚的例子中比如一个Car类，只要有speak方法，就可以实现多态。

Python的多态最大的特点是：鸭子类型（Duck Typing）。“如果它走起路来像鸭子，叫起来像鸭子，那么它就是鸭子。”

5.2 多态的应用

5.2.1 方法重写（最常见的形式）

5.2.2 运算符重载

相同的运算符（如 +, *）对不同类型的对象有不同的行为。

5.2.3 内置函数的多态

像 len() 这样的函数可以对多种类型的对象工作。

5.3 抽象类（接口）

在Python中，抽象类和接口是一个东西，方法体是空实现的（pass）称之为抽象方法，含有抽象方法的类称之为抽象类。

  • 父类用来确定有哪些方法（父类制定接口标准）
  • 具体的方法实现有子类来实现（子类实现接口标准）

在Python中，抽象类的子类并不是必须要实现所有的抽象方法。

6 其他特性

6.1 实例属性和实例方法

使用类名()创建对象，创建对象的动作有两步：

  1. 在内存中为对象分配空间
  2. 调用初始化方法__init__为对象初始化

对象创建后，内存中就有了一个对象的实实在在的存在——实例。

  • 创建出来的对象叫做类的实例，创建对象的动作叫做实例化
  • 对象的属性叫做实例属性，对象的方法叫做实例方法

对象各自拥有自己的实例属性，可以通过self. 访问自己的属性、调用自己的方法。

每一个对象都有自己独立的内存空间，保存各自不同的属性。 多个对象的方法，在内存中只有一份，调用方法时，需要把对象的引用传递到方法的内部。

6.2 类属性和类方法

  • 在程序运行时，类同样会被加载到内存，在Python中，类是一个特殊的对象——类对象。
  • 在程序运行时，类对象在内存中只有一份，使用一个类可以创建出很多个对象的实例。

除了封装实例的属性和方法外，类对象还可以拥有自己的属性和方法——类属性、类方法。

  • 类属性：给类对象定义的属性，与具体对象无关，用来记录类对象的相关特征
  • 类方法：给类对象定义的方法，可以访问类对象或调用其他类方法

类方法第一个参数必须为类对象，通常用cls表示，类似实例方法中的self，不需要传值。 在类方法中，哪个类调用的方法，cls就是哪个类的引用，可以通过cls.访问类属性或调用类方法。

6.3 静态方法

既不需要访问实例属性和实例方法，又不需要访问类属性和类方法，就可以使用静态方法，使用装饰器@staticmethod来标识。

  • 静态方法：类似于独立的函数，只是放在类中组织管理，不能直接访问或修改类的状态。不依赖于类或实例状态，适合包含与类无关的功能。
  • 类方法：绑定到类，可以访问和修改类的状态。依赖于类的状态，适合需要访问类属性或影响类行为的场景。
