07_Python生成器和迭代器

1 迭代

for循环这种遍历方式叫做迭代（Iteration）。

2 推导式（生成式）

推导式可以用来生成列表、集合、字典。

3 生成器

受到内存限制，列表的容量是有限的，存储大量数据要占用大量的空间。

生成器解决了这个问题，生成器（generator）允许按需生成值，而不是一次性在内存中创建并存储整个序列。
在每次请求时（使用 next() 或在 for 循环中）才计算并返回一个值。计算完一个值后，它会“暂停”执行，保存当前的状态（包括局部变量），直到下一次请求到来时再“唤醒”并从暂停的地方继续执行。这种方式被称为惰性求值。

3.1 生成器表达式

生成器表达式在语法上类似于列表推导式，把 []改成 ()就创建了一个 generator。

  • 列表推导式：立即生成所有元素，并存储在内存中。
  • 生成器表达式：返回一个生成器对象，按需生成元素。

3.2 生成器函数

推导算法复杂的时候可以使用函数实现，这是最常见的方法，写法和普通函数类似，不过使用 yield 语句而不是 return 语句来返回值。

  • return：函数遇到 return 会立即终止，并返回一个值。每次调用函数都会从第一行代码重新开始执行。
  • yield：函数遇到 yield 会暂停执行，将值返回给调用者，并记住函数当前的所有状态。下次调用时，函数会从上次返回的 yield语句继续执行。

如果一个函数定义中包含 yield关键字，那么这个函数就不再是一个普通函数，而是一个 generator函数。

注意：调用 generator函数会创建一个 generator对象，多次调用 generator函数会创建多个相互独立的 generator，所以正确做法是创建一个 generator对象。

3.3 生成器的访问

如果要访问生成器，可以使用 next()函数。generator保存的是算法，每次调用 next()就计算 g的下一个值，直到计算到最后一个元素，没有更多的元素时，抛出 StopIteration的错误。

不过一般不会使用 next()，由于 generator也是可迭代对象，可以通过 for循环去迭代它，并且不用担心相关 StopIteration的错误。

4 迭代器

4.1 可迭代对象

可以直接作用于 for循环的对象统称为可迭代对象（Iterable），包括：

  • 集合数据类型（都是 Iterable，但不是 Iterator，可以使用 iter()函数转化为迭代器）
    • 字符串
    • 列表
    • 元组
    • 字典
    • 集合
  • 生成器（都是 Iterable，又是 Iterator）
    • 生成器表达式
    • 生成器函数
  • range序列

可以使用 isinstance方法判断是否为可迭代对象 Iterable。

4.2 迭代器对象

生成器不但可以作用于 for循环，还可以被 next()函数不断调用并返回下一个值，直到最后抛出 StopIteration错误表示无法继续返回下一个值了。
迭代器是一个实现了 __iter__() 和 __next__() 方法的对象，可以被 next()函数调用并不断返回下一个值的对象称为迭代器（Iterator）。

也可以使用 isinstance方法判断是否为迭代器对象 Iterator。

生成器都是 Iterator对象，但 list、dict、str虽然是 Iterable，却不是 Iterator。
把 list、dict、str等 Iterable变成 Iterator可以使用 iter()函数：

  • 凡是可作用于 for循环的对象都是 Iterable类型
  • 凡是可作用于 next()函数的对象都是 Iterator类型，它们表示一个惰性计算的序列

为什么 list、dict、str等数据类型不是 Iterator？
这是因为Python的 Iterator对象表示的是一个数据流，Iterator对象可以被 next()函数调用并不断返回下一个数据，直到没有数据时抛出 StopIteration错误。可以把这个数据流看做是一个有序序列，但我们却不能提前知道序列的长度，只能不断通过 next()函数实现按需计算下一个数据，所以 Iterator的计算是惰性的，只有在需要返回下一个数据时它才会计算。
Iterator甚至可以表示一个无限大的数据流，例如全体自然数。而使用list是永远不可能存储全体自然数的。

Python的 for循环本质上就是通过不断调用 next()函数实现的。
